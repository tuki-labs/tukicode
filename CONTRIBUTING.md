# Contributing to TukiCode

First off, thank you for considering contributing to TukiCode! It's people like you that make TukiCode such a great tool for developers.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.
* Use the GitHub issue search — check if the issue has already been reported.
* Check if the issue has been fixed — try to reproduce it using the latest `main` branch.
* Create a new issue using a clear and descriptive title.

### Suggesting Enhancements
* Open a new issue and explain your enhancement.
* Provide specific examples to demonstrate the steps and the expected behavior.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`pytest`).
5. Make sure your code is properly formatted (we use standard Python style guides).
6. Issue that pull request!

## Local Development Setup
1. Clone your fork locally.
2. Install dependencies: `pip install -r requirements.txt`
3. Run TukiCode locally: `python tuki.py`

## Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. Let's build a welcoming, inclusive, and respectful community!
