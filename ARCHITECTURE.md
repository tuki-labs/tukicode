# TukiCode Architecture

This document details the internal structure, the functions of each file, and how the components interact to bring the TukiCode agent to life.

## Project Structure

### Project Root
- **tuki.py**: Main entry point (CLI). Manages top-level commands (chat, config, history, models) using the Typer library.
- **config.py**: Centralized configuration system. Loads and saves preferences from tukicode.toml.
- **agent_icon.py**: Contains the ASCII art and animation logic for the Tuki mascot.

### Core Controller (/core)
- **controller.py**: The TukiController class is the central orchestrator. It manages the MVC separation by holding the business logic, database history, AI model switching, and API key management independently from the Textual UI.

### Agent Core (/agent)
The intelligence engine resides here:
- **loop.py**: The heart of the agent. Implements the ReAct (Reason + Act) pattern asynchronously. Orchestrates sending prompts to the LLM, executing tools, and updating context.
- **context.py**: Manages the short-term memory of the conversation. Includes logic for token counting and history compression if the context window is exceeded.
- **planner.py**: Generates structured step-by-step plans, enforcing JSON-only outputs using response_format.
- **executor.py**: Asynchronously executes the plans generated by the planner, managing step verification and retries.
- **parser.py**: Analyzes model responses to identify when the agent wants to use a tool or provide a final response.
- **ollama_client.py**: Async client to interact with the local Ollama API.
- **openrouter_client.py**: Async client (httpx) to interact with the OpenRouter API (cloud models).
- **anthropic_client.py**: Async client using AsyncAnthropic.
- **gemini_client.py**: Async client using Google's GenerativeAI async methods.

### User Interface (/ui)
The UI is built on an event-driven architecture using Textual:
- **app.py**: The Main App. Defines the layout, CSS styles, and handles user events. It communicates strictly with the TukiController.
- **display.py**: Abstraction layer acting as a bridge between the AgentLoop and the TukiApp. Ensures messages are rendered correctly in the appropriate thread.

## Visual Design and UI (Textual)

TukiCode uses Textual (a Python-based TUI framework) to offer a premium and stable experience.

### Layout Components
1.  **Header**: Displays the app title and a clock.
2.  **Left Panel (#left-panel)**: A Static widget showing the Tuki animation using an update interval.
3.  **Chat Log (#chat-log)**: A RichLog widget managing message history with smooth scrolling and Markdown rendering.
4.  **Thinking Panel (#thinking-panel)**: A dynamically toggled panel above the input bar to display the agent's reasoning.
5.  **Input Bar (#input-bar)**: Text field where the user types.
6.  **Status Bar (#status-bar)**: Displays real-time information (Model, Tokens, Risk Level).

### How does communication work?
Because Textual manages its own event loop, the agent (AgentLoop) runs in separate asynchronous tasks. To update the UI without causing thread errors:
1.  The agent calls display.show_message().
2.  display.py uses app.call_later(app.add_message, ...) to schedule the update in the main UI thread.
3.  This ensures the terminal does not break and the interface remains reactive.

## Extending the UI

### 1. Adding new Widgets
To add elements (e.g., a list of open files), edit the compose() method in ui/app.py:
```python
def compose(self) -> ComposeResult:
    yield Header()
    with Horizontal():
        yield Static(id="my-new-panel") # Add this
        yield RichLog(id="chat-log")
    # ...
```

### 2. Changing Styles (CSS)
Textual uses a syntax similar to CSS3. You can modify TukiApp.CSS in ui/app.py to change colors, borders, or visibility:
```css
#my-new-panel {
    background: #111;
    border: solid #333;
    width: 20;
}
```

### 3. Adding Slash Commands
New commands like /stats should be added in TukiApp.handle_command():
```python
elif text == "/stats":
    self.add_message("system", f"Tokens used: {self.context.token_count}")
```

## Key Technologies
- **Textual**: Main framework for layout and TUI event handling.
- **Rich**: For Markdown rendering, ANSI colors, and tables.
- **Typer**: For creating the CLI command interface.
- **SQLite**: For persistent history storage.
- **httpx / asyncio**: For high-performance, non-blocking network requests.
